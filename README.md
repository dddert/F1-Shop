# f1_shop

`f1_shop` — учебное веб-приложение на Flask для интерактивной презентации гоночного болида Formula 1 и оформления закрытого предзаказа.

Проект реализован в рамках учебной ознакомительной практики. Основная цель проекта — изучение современных веб-технологий, серверной обработки данных, работы с шаблонами, формами, JSON-хранилищем и 3D-визуализацией в браузере.

## Описание проекта

Приложение представляет собой сайт-презентацию болида `NERO BLITZ F1-26`.

Пользователь может:

- просмотреть основную информацию о болиде;
- изучить технические характеристики;
- взаимодействовать с 3D-моделью;
- переключать цвет болида;
- открыть страницу закрытого предзаказа;
- отправить заявку через форму.

Администратор может открыть отдельную панель и просмотреть список поступивших заявок.

## Функциональность

Основные возможности приложения:

- главная страница с презентацией болида;
- отображение 3D-модели в формате GLB;
- интерактивные точки на 3D-модели;
- изменение цвета модели через JavaScript;
- страница оформления предзаказа;
- отправка формы методом POST;
- обработка JSON-запросов на сервере;
- сохранение заявок в файл `orders.json`;
- административная панель для просмотра заявок;
- расчет количества заявок и потенциальной суммы заказов.

## Назначение основных файлов

### `app.py`

Главный файл Flask-приложения.

В нем реализованы:

* создание объекта приложения;
* маршруты сайта;
* передача данных в HTML-шаблоны;
* обработка формы предзаказа;
* чтение и запись заявок в JSON-файл;
* вывод административной панели.

### `templates/index.html`

Главная страница проекта.

Содержит:

* описание болида;
* технические характеристики;
* блок с 3D-моделью;
* интерактивные точки;
* кнопки смены цвета;
* переход к странице предзаказа.

### `templates/preorder.html`

Страница формы предзаказа.

Содержит поля для ввода данных пользователя и JavaScript-код для отправки заявки на сервер.

### `templates/admin.html`

Административная панель.

Отображает список заявок из файла `orders.json`, количество заказов и расчет потенциальной суммы.

### `orders.json`

Файл для хранения заявок пользователей.

Данные сохраняются в формате JSON.

### `static/models/f1.glb`

3D-модель болида, которая отображается на главной странице через компонент `model-viewer`.

## Маршруты приложения

| Маршрут         | Метод | Назначение                     |
| --------------- | ----- | ------------------------------ |
| `/`             | GET   | Главная страница               |
| `/preorder`     | GET   | Страница оформления предзаказа |
| `/api/preorder` | POST  | Прием данных формы             |
| `/admin-panel`  | GET   | Административная панель        |

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/username/f1_shop.git
cd f1_shop
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
```

### 3. Активация виртуального окружения

Для Windows:

```bash
venv\Scripts\activate
```

Для Linux/macOS:

```bash
source venv/bin/activate
```

### 4. Установка зависимостей

```bash
pip install flask
```

### 5. Запуск приложения

```bash
python app.py
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:5000/
```

## Пример работы приложения

1. Пользователь открывает главную страницу.
2. Flask загружает шаблон `index.html`.
3. На странице отображаются данные о болиде и 3D-модель.
4. Пользователь переходит на страницу предзаказа.
5. Пользователь заполняет форму.
6. JavaScript отправляет данные на маршрут `/api/preorder`.
7. Flask принимает данные и сохраняет их в `orders.json`.
8. Администратор открывает `/admin-panel`.
9. Заявки отображаются в виде таблицы.

## Особенности реализации

В проекте используется простое файловое хранилище на основе `orders.json`. Такой подход подходит для учебного проекта, так как он прост в реализации и позволяет наглядно показать процесс сохранения данных.

Файл `database.db` присутствует в структуре проекта, но в текущей версии основная логика хранения заявок реализована через JSON-файл. В дальнейшем проект может быть доработан с использованием SQLite.
